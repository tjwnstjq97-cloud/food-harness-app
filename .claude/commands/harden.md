현재 하네스 구조를 강화해.

목표:
- 반복 실수 방지
- 보안 강화
- validator 체계 개선
- 실패 기록 체계 점검

규칙:
1. 추가/수정 작업은 TASK 단위로 독립 수행한다
2. 각 TASK는 최대 3회까지 재시도한다
3. 3회 실패 시 중단하고 failed_tasks.md에 기록한다
4. 실제 .env 파일은 절대 생성하거나 수정하지 말 것
5. research.md에 강화 내용과 남은 TODO를 기록할 것

출력 형식:
- 강화한 항목
- 추가한 validator 또는 규칙
- 실패한 항목
- 생성/수정 파일 목록
- 다음 추천 강화 작업
